How to Run a Free Scan

Step 1 — Open the Scanner

Go to edgeiqlabs.com and scroll to the free scanner section, or select a tool directly from the navigation.

Step 2 — Enter Your Target

Type the URL or domain you want to scan. For subdomain enumeration, enter your base domain (e.g. example.com). For XSS, enter the target URL with a parameter (e.g. https://example.com/search?q=test).

Step 3 — Run the Scan

Click Scan. Results appear instantly. No data is stored — results are shown in your browser only.

Step 4 — Interpret Results

Each scanner returns structured findings with risk ratings, technical details, and remediation recommendations. Pro users get exportable JSON/PDF reports.

Method 2 — OpenClaw Skills

OpenClaw is a local AI agent platform. EdgeIQ tools are published as OpenClaw skills, which means you can run them from your terminal, automate them on a schedule, or pipe results into other tools. Requires OpenClaw installed on your machine.

Prerequisites

  • OpenClaw — Install from openclaw.ai
  • Python 3.8+ — Required for most skill scripts
  • API keys — Some tools require free API keys (see individual skill docs)

Install OpenClaw

# Install OpenClaw globally via npm
npm install -g openclaw

# Or via curl (recommended)
curl -fsSL https://openclaw.ai/install.sh | bash

# Verify installation
openclaw --version

Find EdgeIQ Tools on ClawHub

All EdgeIQ tools are published on ClawHub — the skill marketplace for OpenClaw. Search "edgeiq" to see all available tools.

Install an EdgeIQ Skill

# Search for EdgeIQ skills
openclaw skills search edgeiq

# Install a specific skill
openclaw skills install edgeiq-xss-scanner

# Install all EdgeIQ tools at once
openclaw skills install edgeiq-alerting-system edgeiq-network-scanner edgeiq-ssl-watcher

Run a Tool via OpenClaw

# Run directly from terminal
openclaw run edgeiq-xss-scanner --target https://example.com/search?q=test

# Run with output formatting
openclaw run edgeiq-ssl-watcher --domain example.com --format json

# Run with verbose output
openclaw run edgeiq-subdomain-hunter --domain example.com --verbose

Configure API Keys

Some EdgeIQ skills require external API keys. These are set once in your OpenClaw config:

# Set an API key for a skill
openclaw config set edgeiq-ssl-watcher API_KEY your_key_here

# View current config for a skill
openclaw config show edgeiq-ssl-watcher

Automate with Cron

Schedule scans to run automatically and receive alerts via Slack, Discord, or Telegram:

# Schedule a weekly SSL check
openclaw schedule add --name "Weekly SSL Check" \
  --cron "0 9 * * 1" \
  --skill edgeiq-ssl-watcher \
  --args "--domain example.com" \
  --notify slack

Install Alerting (Recommended)

Connect EdgeIQ tools to Slack, Discord, or Telegram for automated alerts:

# Install the alerting system
openclaw skills install edgeiq-alerting-system

# Configure a Discord webhook
openclaw config set edgeiq-alerting-system \
  DISCORD_WEBHOOK https://discord.com/api/webhooks/your-webhook-here

# Test the alerting setup
openclaw run edgeiq-alerting-system --test

Method 3 — Self-Hosted Cloudflare Workers

If you want to run your own instance of any EdgeIQ tool — for higher rate limits, white-labeling, or custom integrations — every tool is available as a Cloudflare Worker you can deploy to your own account.

Prerequisites

  • Cloudflare account — Free at cloudflare.com
  • Wrangler CLI — Cloudflare's Worker deployment tool

Install Wrangler

# Install Wrangler via npm
npm install -g wrangler

# Or via pip
pip install wrangler

# Verify
wrangler --version

Deploy a Worker

# Clone the Worker source (example: SSL Checker)
git clone https://github.com/snipercat69/edgeiq-labs
cd edgeiq-labs/edgeiq-ssl-checker

# Configure your Cloudflare account
wrangler login

# Deploy
wrangler deploy

# Your Worker URL will be returned, e.g.:
# https://edgeiq-ssl-checker.your-subdomain.workers.dev

Configure Environment Variables

Workers often use external APIs. Set these in wrangler.toml or via Cloudflare Dashboard:

# In wrangler.toml
[vars]
ALLOWED_ORIGIN = "https://yourdomain.com"

# For API keys, use secrets
wrangler secret put CRT_SH_API_KEY
# Enter your API key when prompted

Bind to Your Custom Domain

# In wrangler.toml, add a route
routes = [
  { pattern = "ssl.yourdomain.com", zone_name = "yourdomain.com" }
]

# Redeploy
wrangler deploy
